# 数据源代码目录规则

## 目录结构

```text
src/ashare_review/collectors/
├── contracts.py        # 公共采集接口
├── http.py             # HTTPS、超时与传输层
├── resilience.py       # 限速与重试
├── runner.py           # 采集运行与健康记录
├── manual_csv.py       # 本地 CSV 导入，不属于网站适配器
└── sources/
    ├── sse.py          # 上海证券交易所
    ├── szse.py         # 深圳证券交易所
    ├── cninfo.py       # 巨潮资讯
    └── csrc.py         # 中国证监会
```

## 强制规则

1. 一个网站只能对应 `sources/` 下的一个 Python 文件，文件名采用网站的稳定英文简称。
2. 某网站的 URL、请求参数、页面或响应解析、字段映射、来源 ID 和网站特有错误处理，只能写在该网站自己的文件中。
3. 不得在一个文件中混写两个网站的抓取代码，也不得从一个网站适配器直接调用另一个网站适配器。
4. HTTPS、超时、限速、重试、健康记录、公共数据模型等通用能力放在 `collectors/` 根目录，不能复制到各网站文件中。
5. 新增网站时，必须先完成数据源准入记录，再新增一个同名或稳定简称的文件，例如 `eastmoney.py`；不修改已有网站适配器的职责边界。
6. 本地文件导入使用 `manual_csv.py`，它不代表任何网站，也不得放入 `sources/`。

## 当前状态

`sse.py`、`szse.py`、`cninfo.py` 和 `csrc.py` 目前只声明各自负责的来源 ID，尚未实现真实网络访问。后续实现必须遵守本文件的分文件规则和数据源有条件批准范围。
