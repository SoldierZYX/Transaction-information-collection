# 新闻事件处理规则

## 处理顺序

1. 读取已落库且 `record_type` 为 `news` 的原始记录。
2. 对标题执行 Unicode NFKC 规范化、空白合并和大小写归一。
3. 使用上海业务日和标准化标题生成稳定事件键。
4. 只将同一业务日且标题完全相同的记录归并为一个事件。
5. 为每个事件写入所有原始记录的 `event_evidence` 关联。

## 当前边界

- 事件类别固定为 `news`，方向固定为 `neutral`。
- 不从标题推断利好、利空、行业或证券关联。
- 不使用模糊匹配、语义相似度或 AI 合并事件。
- 重复运行只更新既有事件，不会重复创建证据关联。

这些限制用于避免仅凭新闻元数据产生不可审计的市场结论。后续若增加分类或关联规则，必须记录规则版本和证据依据。
