# 数据使用与合规检查清单

在批准任一数据源、真实网络访问或付费 API 前，负责人必须逐项确认。

- [ ] 访问方式符合提供方服务条款、官方文档和 robots 规则。
- [ ] 已确认数据是否允许研究、存储、内部使用和再展示。
- [ ] 不使用绕过登录、验证码、访问控制或频率限制的方式。
- [ ] 已配置保守限速、超时、有限重试和失败记录。
- [ ] 仅保存实现审计所需的最少数据；不批量保存受版权保护的正文。
- [ ] 原始响应、日志和报告不含 API Key、SMTP 授权码或其他密钥。
- [ ] 数据源有稳定的来源 ID、证据链接、抓取时间与去重策略。
- [ ] 已确认发布日期、交易日期和时区的字段语义。
- [ ] 已登记字段变化、服务中断、授权变更和数据延迟的降级策略。
- [ ] 真实网络测试经过人工确认，默认采用 mock 测试。
- [ ] 输出中包含研究用途与非投资建议提示。
- [ ] AI 仅处理已筛选证据，结构化结果校验失败时不得入库。
